# Лендинг «Золотой Молот»

Одностраничный адаптивный сайт-презентация вымышленного продукта «Золотой Молот».

Сайт реализован на **React + Vite + Tailwind (через Vite-плагин)** с дополнительными файлами `index.html`, `styles.css` и `script.js`, чтобы его можно было открыть как обычный статический HTML.

## Возможности лендинга

- Современный тёмный дизайн с золотым акцентом.
- Секции: Hero, Наше решение / Ваша проблема, Функции, Как это работает, Цены / Заказ, Отзывы, FAQ, Контакты.
- Липкий хедер с навигацией и мобильным меню-гамбургером.
- Адаптивная вёрстка под мобильные устройства.
- Плавные анимации появления секций при скролле (`IntersectionObserver`).
- Мини-демо механики работы молота (удар → предмет чинится → улучшается).
- Фейковая корзина с выбором тарифа, количества и подарочной упаковки.
- Модальное окно «оплаты» с валидацией имени и email.
- Экран подтверждения с фейковым номером заказа.
- Сохранение демо-заказов в `localStorage` и отображение истории.
- Лёгкая анимация конфетти при успешной «покупке».
- SEO meta + Open Graph теги.

## Структура проекта

- `index.html` — HTML-шаблон, подключает React-приложение (`src/main.tsx`), `styles.css` и `script.js`.
- `styles.css` — дополнительные глобальные стили, анимации (hero, конфетти, появления блоков, демо-удар молота).
- `script.js` — вспомогательный JS для плавного скролла и анимаций в чистом HTML-режиме (без React).
- `assets/` — заглушки изображений и иконок.
  - `golden-hammer.png` — иллюстрация молота (можете заменить своей картинкой, сохранив имя файла).
  - `og-golden-hammer.svg` — Open Graph изображение для соцсетей.
  - `favicon.svg` — иконка сайта.
- `src/` — React-приложение.
  - `main.tsx` — точка входа React.
  - `App.tsx` — основное приложение-лендинг со всеми секциями и логикой.
  - `index.css` — подключение Tailwind через Vite.
  - `utils/cn.ts` — утилита объединения классов.

## Как запустить локально (вариант с Vite)

1. Установите зависимости:

   ```bash
   npm install
   ```

2. Запустите dev-сервер Vite:

   ```bash
   npm run dev
   ```

3. Откройте адрес, указанный в консоли (обычно `http://localhost:5173`).

## Как открыть как статический HTML

Собранный проект можно открыть как обычный сайт:

1. Выполните сборку:

   ```bash
   npm run build
   ```

2. В директории `dist/` будет готовый `index.html` (благодаря `vite-plugin-singlefile` всё собрано в один файл). Его можно открыть двойным кликом в браузере или раздавать с любого статического хостинга.

На этапе разработки можно также открыть корневой `index.html` напрямую в браузере или через плагин **Live Server** — часть интерактива будет доступна за счёт `script.js`, но полный функционал (форма заказа, модальные окна и т.д.) рассчитывается на работу через Vite/React.

## Замена текстов и изображений

- Тексты всех секций находятся в `src/App.tsx`. Ищите нужные заголовки/абзацы и меняйте строки.
- Иллюстрация молота задаётся через путь `/assets/golden-hammer.png` в компоненте `Hero`. Замените файл картинкой такого же имени.
- Иконки соцсетей и ссылки в футере также можно скорректировать в `src/App.tsx`.

## Сторонние библиотеки

- **React / ReactDOM** — рендер и компоненты.
- **Vite** — сборка и dev-сервер.
- **Tailwind CSS 4 (через `@tailwindcss/vite`)** — утилитарные классы стилей.
- **tailwind-merge, clsx** — объединение и мердж классов.
- Дополнительные анимации и логика реализованы на чистом TypeScript/JavaScript без сторонних анимационных библиотек (GSAP и т.п.).

## Доступность

- Семантическая разметка (`header`, `main`, `section`, `footer`).
- ARIA-атрибуты для модальных окон, мобильного меню и иконок.
- Фокус-стили и управление с клавиатуры для кнопок и интерактивных элементов.

## Где поправить адаптивность

Основная адаптивность реализована через классы Tailwind в JSX. Дополнительные медиазапросы можно добавить в `styles.css` (раздел `Базовая адаптивность`).
